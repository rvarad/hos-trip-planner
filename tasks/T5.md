# T5: Geo provider stack + routing integration

**Goal:** Choose the geo stack (map rendering, geocoding/autocomplete, routing),
record it in an ADR, and integrate a free routing API to get driving distance and
duration between the trip's coordinates.

**Chosen stack (pure OSS, no API key):**
- Map rendering: **MapLibre GL JS** with free vector tiles (e.g. OpenFreeMap).
- Geocoding + autocomplete + reverse: **Photon** (Komoot). Start against the
  public `photon.komoot.io` instance; endpoint is swappable via env var
  (`PHOTON_URL`) so it can be self-hosted later (see T12).
- Routing: **OSRM** (or OpenRouteService) — coordinates in, distance/duration out.

## Subtasks

- [ ] **T5.1** ADR: geo provider stack
  - Add `docs/adr/002-geo-stack.md` recording the MapLibre GL / Photon / OSRM
    choices, the "public now, self-host later" decision for Photon, and the

- [ ] **T5.2** Routing client

  - Test: stub/mock the HTTP call; assert distance and duration are parsed and
