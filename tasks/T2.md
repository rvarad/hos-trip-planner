# T2: HOS engine data shapes + function skeleton

**Goal:** Define the input/output data structures and skeleton function signatures
for the HOS engine. No logic yet — just types and stubs that make the test file
importable and the function contracts clear.

## Subtasks

- [ ] **T2.1** Define `TripInput` dataclass in `engine.py`
  - Fields: `current_location: str`, `pickup: str`, `dropoff: str`,
    `cycle_hours_used: float`, `start_time_minutes: int`
  - Test: import `TripInput`, instantiate with sample values, assert field access works.

- [ ] **T2.2** Define `Segment` dataclass in `engine.py`
  - Fields: `kind: str` (driving/on_duty/off_duty/sleeper), `start: int`,
    `duration: int` (minutes from trip start)
  - Test: instantiate, assert `kind` and `duration` accessible.

- [ ] **T2.3** Define `DayLog` dataclass in `engine.py`
  - Fields: `date_offset: int`, `segments: list[Segment]`
  - Test: construct a `DayLog` with one segment, assert `date_offset` and
    `segments[0].kind` accessible.

- [ ] **T2.4** Define `PlanResult` dataclass in `engine.py`
  - Fields: `segments: list[Segment]`, `days: list[DayLog]`, `total_miles: float`
  - Test: instantiate with empty lists, assert fields accessible.

- [ ] **T2.5** Stub `plan_trip(inp: TripInput) -> PlanResult` in `engine.py`
  - Raises `NotImplementedError` for now.
  - Test: call with a valid `TripInput`, assert `NotImplementedError` is raised.

- [ ] **T2.6** Define HOS constants in `rules.py`
  - `MAX_DRIVING_MINUTES`, `DRIVING_WINDOW_MINUTES`, `BREAK_AFTER_MINUTES`,
    `BREAK_DURATION_MINUTES`, `CYCLE_HOURS`, `CYCLE_DAYS`, `REST_RESET_MINUTES`
  - Test: import each constant, assert it equals its expected integer value.
