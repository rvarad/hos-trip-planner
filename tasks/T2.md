# T2: HOS engine data shapes + function skeleton

**Goal:** Define the input/output data structures and skeleton function signatures
for the HOS engine. No logic yet — just types and stubs that make the test file
importable and the function contracts clear.

> **Note — shapes reconciled before T3 (2026-06-01).** A pre-T3 audit replaced the
> shapes below with a richer, leg-based model so the engine has the data it needs
> to schedule (per-leg miles/minutes) and to label segments. The subtasks here are
> kept as the historical record of what T2 originally delivered; the engine now
> reflects the reconciled model:
> - `TripInput`: location-triple → `legs: list[RouteLeg]`; `cycle_hours_used`
>   (hours) → `current_cycle_used_minutes` (minutes, converted at the T4 boundary).
> - **New** `RouteLeg` (routing input: start/end `Location`, `distance_miles`,
>   `duration_minutes`, pass-through `geometry`) and `DriverState` (mutable running
>   clocks + `miles_since_fuel`).
> - `Segment` (free-string `kind`) → `DutySegment` (`start_min`/`end_min`,
>   `status: DutyStatus` enum, `description`, start/end `Location`, `miles`).
> - HOS constants locked to the FMCSA handbook with § citations (see `rules.py`).

## Subtasks

- [x] **T2.0** Define `Location` dataclass in `engine.py`
  - Fields: `label: str`, `lat: float`, `lng: float`.
  - Rationale: locations carry coordinates because the frontend resolves every
    location to a point — either by geocoding typed text or by a dropped map pin
    (see T6b). The endpoint passes coords to the routing API (T5).
  - Test: instantiate with sample values, assert `label`/`lat`/`lng` accessible.

- [x] **T2.1** Define `TripInput` dataclass in `engine.py`
  - Fields: `current_location: Location`, `pickup: Location`, `dropoff: Location`,
    `cycle_hours_used: float`, `start_time_minutes: int`
  - Test: import `TripInput`, instantiate with sample `Location` values, assert field access works.

- [x] **T2.2** Define `Segment` dataclass in `engine.py`
  - Fields: `kind: str` (driving/on_duty/off_duty/sleeper), `start: int`,
    `duration: int` (minutes from trip start)
  - Test: instantiate, assert `kind` and `duration` accessible.

- [x] **T2.3** Define `DayLog` dataclass in `engine.py`
  - Fields: `date_offset: int`, `segments: list[Segment]`
  - Test: construct a `DayLog` with one segment, assert `date_offset` and

- [x] **T2.4** Define `PlanResult` dataclass in `engine.py`
  - Fields: `segments: list[Segment]`, `days: list[DayLog]`, `total_miles: float`
  - Test: instantiate with empty lists, assert fields accessible.

- [x] **T2.6** Define HOS constants in `rules.py`
