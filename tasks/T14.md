# T14: Log the 10-hour rest as sleeper berth

**Goal:** Stamp the 10-hour driving reset with `SLEEPER_BERTH` instead of
`OFF_DUTY`, so the ELD log sheet's sleeper row reflects how over-the-road
drivers actually log their overnight rest. Until now the sleeper row was always
empty.

## Decision (agreed 2026-06-01)

- **10-hour reset → `SLEEPER_BERTH`.** The overnight rest is the clear sleeper case.
- **34-hour restart → stays `OFF_DUTY`.** Typically "home time", logged off-duty.
- **30-min break → stays `OFF_DUTY`.** Drivers log short breaks off-duty, not in
  the berth; flipping it would be less realistic.
- **Pre/post-trip + gap fill → stays `OFF_DUTY`.** This is `DailyLogSheet`'s
  auto-fill of grid time the engine timeline doesn't cover (before the trip
  starts, after drop-off). Unchanged.
- **Not** modeling split-sleeper provisions (§ 395.1(g)) — still out of MVP scope.
- Compliance is unaffected: a 10h reset off-duty is equally valid; this only
  changes the status label, which an ELD reader expects to be sleeper.
