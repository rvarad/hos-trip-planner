# T3: HOS engine logic + tests from FMCSA examples

**Goal:** Implement `plan_trip` in `engine.py` so it produces an HOS-compliant
timeline (`list[DutySegment]`) against the leg-based model from the pre-T3
reconcile. All limits in `rules.py` (FMCSA §395) must be enforced. Tests use the
worked examples from the FMCSA guide
(`reference/fmcsa-hos-395-drivers-guide-to-hos-2022-04-28-0-1-.pdf`) as ground truth.

## Design (agreed 2026-06-01)

The engine walks `TripInput.legs` in order, emitting `DutySegment`s and updating a
mutable `DriverState`:

- **Time origin:** driver goes on duty at minute 0; segments are minutes-from-trip-start.
  Wall-clock comes from `start_time_minutes` only at the edges. Day slicing into
  `DayLog`s is **T9** — T3 returns `days=[]` and fills `segments` + `total_miles`.
- **Cycle seed:** `on_duty_in_cycle` starts at `current_cycle_used_minutes`.
- **Per leg:** spend `duration_minutes` of `DRIVING`, advancing `miles_since_fuel`
  by `distance_miles`. Split the driving to insert non-driving time when a cap binds.
- **Pickup / drop-off:** 60-min `ON_DUTY_NOT_DRIVING` after the first leg (pickup)
  and after the last leg (drop-off).

Key semantics:

- **30-min break (§395.3(a)(3)(ii)):** satisfied by *any* non-driving segment
  ≥ `MIN_BREAK_MIN` — off-duty, sleeper, OR on-duty-not-driving. So a fuel stop,
  pickup, or drop-off resets `driving_since_break`; a dedicated break is inserted
  only when `DRIVING_BEFORE_BREAK_MIN` is reached with no qualifying stop since.
- **14h window (§395.3(a)(2)) does NOT pause:** breaks, fuel, and pickup/drop-off
  all consume `elapsed_in_window`; only a 10h reset clears it.
- **Mid-leg stop location:** linearly interpolate lat/lng between the leg's
  endpoints by the fraction of the leg driven, labelled e.g.
  `"En route (~187 mi from Chicago, IL)"`. (T10 may later snap it to the polyline.)
- **70h cap (§395.3(b)):** when remaining cycle is insufficient, insert a 34h
  restart (§395.3(c), `RESTART_OFF_DUTY_MIN`) and reset `on_duty_in_cycle` to 0.

## Subtasks

- [x] **T3.1** Walk the legs (no HOS limits yet)
  - Emit a `DRIVING` segment per leg plus 60-min `ON_DUTY_NOT_DRIVING` at pickup
    (after leg 0) and drop-off (after the last leg). Seed `on_duty_in_cycle`;
    set `total_miles` = sum of leg miles.
  - Test: a 2-leg trip yields `[drive, pickup(60), drive, dropoff(60)]`, segment
    times contiguous from minute 0, `total_miles` equals the leg sum.

- [x] **T3.2** 11-hour driving limit + 14-hour window → 10-hour reset
  - While spending driving, if `driving_in_period` would exceed
    `MAX_DRIVING_PER_PERIOD_MIN` (660) or `elapsed_in_window` would exceed
    `MAX_DUTY_WINDOW_MIN` (840), split the driving and insert a
    `MIN_RESET_OFF_DUTY_MIN` (600) `OFF_DUTY` segment; reset period/window/break clocks.
  - Test: a leg needing >11h driving splits with a 600-min off-duty reset; no
    driving past minute 660 within a period and none past the 14h window.

- [x] **T3.3** 30-minute break after 8h cumulative driving
  - Track `driving_since_break`; when it would exceed `DRIVING_BEFORE_BREAK_MIN`
    (480), insert a `MIN_BREAK_MIN` (30) `OFF_DUTY` break — unless a qualifying
    non-driving segment (fuel/pickup/drop-off/reset) already reset the clock.
  - Test: 9h continuous driving produces a 30-min break at the 8h mark; AND a trip
    where a fuel stop or pickup precedes the 8h mark produces NO separate break.

- [ ] **T3.4** Fuel stop every 1,000 miles (also satisfies the break)
  - Track `miles_since_fuel` (interpolated within a leg). On crossing
    `FUEL_INTERVAL_MILES` (1000), insert a `FUEL_STOP_MIN` (30)
    `ON_DUTY_NOT_DRIVING` fuel stop; reset `miles_since_fuel` and
    `driving_since_break`.
  - Test: a 1,500-mile trip contains exactly one fuel-stop segment, and it clears
    the since-break clock.

- [ ] **T3.5** 70-hour / 8-day cycle cap → 34-hour restart
  - Track `on_duty_in_cycle` (driving + on-duty-not-driving). When remaining cycle
    (`MAX_CYCLE_ON_DUTY_MIN` − used) is insufficient to continue, insert a
    `RESTART_OFF_DUTY_MIN` (2040) `OFF_DUTY` restart and reset `on_duty_in_cycle` to 0.
  - Test: seed 69h, trip needing >1h on-duty → engine inserts a 34h restart and
    continues; on-duty accounting restarts from 0 afterward.

- [ ] **T3.6** Segment locations + descriptions
  - Every `DutySegment` carries `start_location`, `end_location`, and a human
    `description` ("Drive to pickup", "30-min break", "Fuel stop", "10-hour rest",
    "34-hour restart", "Pickup", "Drop-off"). Mid-leg stops use the interpolated
    location described above.
  - Test: a mid-leg fuel stop's location lat/lng lies between its leg's endpoints
    and its description/label reflect the en-route position.

- [ ] **T3.7** FMCSA worked example — 70hr/8-day rolling table
  - Extract a pure cycle-accounting helper; feed the daily on-duty values from the
    guide's 8-day table and assert the rolling total matches the published figure
    for each day.
  - (Out of scope per `rules.py`: sleeper-berth split §395.1(g), adverse driving §395.1(b).)
