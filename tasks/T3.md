# T3: HOS engine logic + tests from FMCSA examples

**Goal:** Implement `plan_trip` in `engine.py` so that it produces an HOS-compliant
segment list. All rules in CLAUDE.md must be enforced. Tests use the worked examples
from the FMCSA guide (`reference/fmcsa-hos-395-drivers-guide-to-hos-2022-04-28-0-1-.pdf`)
as ground truth.

## Subtasks

- [ ] **T3.1** Implement driving-window and 11h driving limit
  - Engine schedules driving segments; stops when either the 11h driving cap or
    the 14h window is reached.
  - Test: trip that fits within one duty period produces no segment past minute 840
    (14h) and no driving past 660 min (11h).

- [ ] **T3.2** Implement mandatory 30-minute break
  - After 8 cumulative driving hours, insert a ≥30 min off-duty/sleeper break.
  - Test: trip of 9h driving generates a break segment before the 8h mark is crossed.

- [ ] **T3.3** Implement 10h off-duty reset
  - After the duty window closes, insert a 10h off-duty segment; clocks reset.
  - Test: a multi-day trip has a ≥600 min off-duty segment between each day's duty
    period, and the next day's driving cap restarts from 0.

- [ ] **T3.4** Implement 70hr/8-day rolling cycle check
  - Seed from `cycle_hours_used`; refuse to drive once the 70h cap is reached.
  - Test: seed with 68h, trip requiring 4h driving — engine halts driving after 2h
    and inserts a rest period before continuing.

- [ ] **T3.5** Implement pickup and drop-off on-duty stops
  - First segment after driving to pickup = 60 min on_duty; same at drop-off.
  - Test: result contains exactly two `on_duty` segments of 60 min each.

- [ ] **T3.6** Implement fuel stop every 1,000 miles
  - Insert a short on-duty fuel stop whenever cumulative miles cross a 1,000-mile
    boundary.
  - Test: 1,500-mile trip contains exactly one fuel-stop segment.

- [ ] **T3.7** FMCSA worked example — 70hr/8-day table
  - Feed the daily on-duty values from the guide's 8-day table, assert the engine's
    rolling total matches the published figures for each day.
