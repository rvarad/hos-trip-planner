# T6: Next.js MUI input form

**Goal:** Build the trip input form: three location pickers (current location,
pickup, drop-off) plus cycle hours used, using MUI components. Each location is
selected by typing (autocomplete) OR dropping a pin — see T6b for the picker
widget. The form's output is three resolved `{ label, lat, lng }` locations plus
`cycle_hours_used`.

## Subtasks

- [ ] **T6.1** Form layout & state
  - Compose three `<LocationField>` (T6b) instances + a cycle-hours MUI field.
  - Hold form state: each location resolves to `{ label, lat, lng } | null`.
  - Test: rendering shows three location fields and the cycle-hours input.

- [ ] **T6.2** Validation & submit
  - Submit disabled until all three locations are resolved and cycle hours valid.
  - On submit, assemble the request body (matching T4's serializer shape) and POST
    via the proxy (T7).
  - Test: submit is blocked with an unresolved location; enabled when all valid.

> Map setup: T5a. Reusable map component: T6a. Picker widget: T6b.
