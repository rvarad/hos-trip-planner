# T7: Next.js server-side proxy routes

**Goal:** App Router route handlers that forward browser requests to the Django
backend and to Photon, keeping the backend internal and the geocoder's endpoint /
usage policy server-side.

## Design (agreed 2026-06-01)

- **Handlers:** `app/api/<name>/route.ts` exporting `GET`/`POST`; Node runtime;
  plain Web `Request`/`Response` (`Response.json`) for easy unit testing.
- **Env (server-side only — keeps the backend internal, no `NEXT_PUBLIC_`):**
  `BACKEND_URL` (default `http://localhost:8000`), `PHOTON_URL`
  (default `https://photon.komoot.io`).
- **plan-trip:** forward the POST body to `${BACKEND_URL}/api/plan-trip`; return the
  upstream **status + JSON verbatim** so validation 400s reach the UI.
- **geocode/reverse:** call Photon with a `User-Agent` header (fair-use) and
  proximity bias, then **transform** Photon GeoJSON features into our clean
  `{ label, lat, lng }` shape via a shared `featureToLocation` helper
  (`geometry.coordinates [lng,lat]` → `lat/lng`; `properties` → a readable label).
  Upstream failure → `502`.
- **Testing:** import each handler, mock `global.fetch` with a canned `Response`,
  assert the upstream URL/headers/params and the returned shape. No network.

## Subtasks

- [x] **T7.1** `/api/plan-trip` proxy
  - `POST` forwards the JSON body to `${BACKEND_URL}/api/plan-trip` and returns the
    upstream status + body unchanged.
  - Test (mocked fetch): the body is forwarded to the backend URL; a 200 passes
    through; a backend 400 propagates as 400.

- [x] **T7.2** `/api/geocode` proxy (autocomplete) + `featureToLocation`
  - `GET ?q=&lat=&lon=` → `${PHOTON_URL}/api?q=…&lat=…&lon=…&limit=…&lang=en` with a
    `User-Agent`. Transform the feature list to `{ label, lat, lng }[]`.
  - Test (mocked fetch): the query + proximity params and `User-Agent` are sent;
    canned Photon GeoJSON is transformed to the clean result list.

- [ ] **T7.3** `/api/reverse` proxy (pin → label)
  - `GET ?lat=&lon=` → `${PHOTON_URL}/reverse?lat=…&lon=…` with a `User-Agent`;
    transform the first feature to `{ label, lat, lng }`.
  - Test (mocked fetch): lat/lon forwarded; the first feature becomes the label.
