# T7: Next.js server-side proxy routes

**Goal:** Implement Next.js route handlers that forward browser requests to backend
services, keeping the backend (and geocoding usage policy / endpoint) internal.

## Subtasks

- [ ] **T7.1** `/api/plan-trip` proxy
  - Forwards POST from the browser to the Django backend at `http://VM-IP:8000`.
  - Test: a POST is forwarded and the backend response is returned through.

- [ ] **T7.2** `/api/geocode` proxy (forward autocomplete)
  - Forwards a query string to Photon's search endpoint, adding the required
    `User-Agent` and passing proximity-bias params. Base URL from `PHOTON_URL`
    env var (defaults to public `photon.komoot.io`, swappable to self-hosted).
  - Test: a query is forwarded with the User-Agent header and results passed back.

- [ ] **T7.3** `/api/reverse` proxy (pin → label)
  - Forwards `lat`/`lng` to Photon's reverse endpoint; same env-var base URL.
  - Test: a reverse request is forwarded and the label returned.

> Photon hosting: public now, self-host later (see T5 ADR and T12).
