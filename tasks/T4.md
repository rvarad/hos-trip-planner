# T4: DRF POST /api/plan-trip

**Goal:** Wire the HOS engine to a DRF endpoint that accepts the trip request
(three coordinate-bearing locations + cycle hours) and returns the JSON timeline.

## Subtasks

- [ ] **T4.1** `LocationSerializer`
  - Fields: `label` (CharField, non-empty), `lat` (FloatField, range [-90, 90]),
    `lng` (FloatField, range [-180, 180]).
  - Test: valid payload passes; out-of-range lat/lng and empty label each raise
    `ValidationError`.

- [ ] **T4.2** `TripRequestSerializer`
  - Nests `LocationSerializer` for `current_location`, `pickup`, `dropoff`;
    plus `cycle_hours_used` (float, 0–70) and `start_time_minutes` (int).
  - Test: a full nested payload validates; a missing location is rejected.

- [ ] **T4.3** `PlanTripView` (POST `/api/plan-trip`)
  - Thin DRF view: validate request → build `TripInput` (with `Location`s) →
    call `engine.plan_trip` → serialize `PlanResult` → return JSON.
  - Test: posting a valid body returns 200 with the timeline shape; an invalid
    body returns 400.
