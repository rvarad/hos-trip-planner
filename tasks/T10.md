# T10: Map display with route + markers

**Goal:** Show the planned route on a map with markers for pickup, drop-off, rest
breaks, and fuel stops. Reuses the `<MapView>` component from T6a (display mode, no
pin picking) so input and result share one map implementation.

## Design (agreed 2026-06-01)

- **Route geometry source:** `build_legs` already returns each `RouteLeg` with a
  `geometry` of `(lat, lng)` points — a full road curve from OSRM, or a 2-point
  straight line from the haversine fallback. The view stitches the legs into one
  polyline and serializes it as `route` (ordered `[lng, lat]` pairs, GeoJSON order).
- **Stops come from the timeline:** every `DutySegment` already carries
  `start_location` / `end_location` and a `description` ("Pickup", "Drop-off",
  "Fuel stop", "30-min break", reset/restart). The frontend derives semantic
  markers from the non-driving segments — no engine or extra backend work.
- **Engine untouched.** T10 is response-shaping (view) + frontend rendering only.

## Subtasks

- [x] **T10.1** Backend: expose route geometry
  - View stitches `legs[*].geometry` into one `[lng, lat]` polyline (dropping the
    duplicated leg-boundary point); new `route` field on
    `PlanTripResponseSerializer`. Always populated (OSRM curve or fallback line).
  - Test: OSRM-mocked response surfaces the road coords in `[lng, lat]` order; the
    fallback response runs current → drop-off as straight legs.

- [x] **T10.2** Frontend: draw the route polyline
  - Optional `route` prop on `<MapView>` rendered as a GeoJSON `Source` + line
    `Layer` (react-map-gl/maplibre). The fallback's 2-point legs render as straight
    lines naturally — no separate empty-route branch needed.
  - Test: MapView renders the line source/layer when given route coords.

- [ ] **T10.3** Frontend: semantic stop markers from the timeline
  - Derive result markers from `planResult.segments` (pickup, drop-off, fuel, rest)
    by mapping `description` → marker `kind` + color (extend `MARKER_COLORS`). Wire
    `TripPlanner`'s map to show route + stops and fit bounds once a plan exists,
    falling back to the three input markers before submit.
  - Test: the segment → marker derivation yields the right kinds/colors.
